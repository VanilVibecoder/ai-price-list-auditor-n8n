# Тест-кейсы

| ID | Сценарий | Вход | Ожидаемый результат |
|---|---|---|---|
| T01 | Корректный CSV | `.csv`, разделитель `;` | Возвращается нормализованный XLSX |
| T02 | Корректный XLSX | `.xlsx` | Возвращается нормализованный CSV |
| T03 | PDF/текст/изображение | неподдерживаемый тип | Telegram сообщает о поддерживаемых форматах |
| T04 | Нет обязательной колонки | отсутствует SKU, название или цена | Mapping gate останавливает обработку |
| T05 | Низкая уверенность AI | `confidence < 0.8` | Telegram сообщает, что структура не определена |
| T06 | Пустой SKU | одна строка без артикула | Строка отклонена с `SKU_EMPTY` |
| T07 | Пустое название | одна строка без названия | Строка отклонена с `NAME_EMPTY` |
| T08 | Цена `по запросу` | нечисловое значение | Строка отклонена с `PRICE_INVALID_FORMAT` |
| T09 | Цена `0` или отрицательная | `0`, `-10` | Строка отклонена с `PRICE_NON_POSITIVE` |
| T10 | Остаток `9800+` | нижняя граница | Экспортируется как `9800`, добавляется `STOCK_LOWER_BOUND` |
| T11 | Остаток `ожидается` | количество неизвестно | Экспортируется с `stock = null`, добавляется `STOCK_VALUE_UNKNOWN` |
| T12 | Точный дубль | одинаковый SKU и одинаковые данные | Первая строка остаётся, копия исключается |
| T13 | Конфликтующий дубль | одинаковый SKU, разные цена/остаток | Все строки группы отклоняются |
| T14 | Нет валидных строк | все строки с ошибками | Telegram сообщает, что ни одна строка не прошла проверку |
| T15 | Ollama недоступна | сервис остановлен | Execution падает; известное ограничение — отдельный Error Workflow не включён |

## Приёмочные критерии

- итоговый файл содержит только `sku`, `name`, `price`, `stock`, `unit`, `manufacturer`, `currency`;
- количество строк в отчёте совпадает с количеством items после парсинга;
- `clean_rows + excluded_rows = total_rows`;
- точные копии SKU не попадают в файл;
- конфликтующие SKU не попадают в файл;
- CSV и XLSX возвращаются в противоположном формате;
- Telegram caption не превышает лимит, заданный в Code node.
